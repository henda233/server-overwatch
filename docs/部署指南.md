## 环境要求

| 项目 | 要求 |
|------|------|
| Python | 3.8+ |
| 系统命令 | `nvidia-smi`, `free`, `top`, `who` |
| 权限 | **普通用户即可**（不需要root） |

### 权限说明

程序仅执行只读系统命令，**不需要root权限**：
- `nvidia-smi` - 查询GPU状态（只读）
- `free -m` - 查询内存（只读）
- `top -bn1` - 查询CPU（只读）
- `who` - 查询在线用户（只读）

## 部署步骤

### 1. 创建专用运行用户

```bash
# 以root登录服务器后执行
sudo useradd -r -s /bin/bash -m -d /home/gpu-monitor gpu-monitor
```

### 2. 安装程序

```bash
# 创建安装目录
sudo mkdir -p /opt/server-overwatch
sudo chown gpu-monitor:gpu-monitor /opt/server-overwatch

# 上传程序文件到 /opt/server-overwatch/
# （可通过scp、rsync或其他方式）

# 安装依赖（以gpu-monitor用户执行）
sudo -u gpu-monitor bash -c "cd /opt/server-overwatch && uv venv .venv"
sudo -u gpu-monitor bash -c "cd /opt/server-overwatch && uv sync"
```

### 3. 配置程序

编辑 `config.yaml`：

```yaml
# QQ机器人配置
qq_bot:
  appid: "你的appid"
  secret: "你的secret"

# 监控配置
monitor:
  interval: 600          # 采集间隔（秒），默认10分钟
  retention_days: 30     # 数据保留天数

# 日志配置
logging:
  level: INFO
  file: "logs/bot.log"
```

## systemd 服务配置

### 1. 创建 Service 文件

在服务器上创建 `/etc/systemd/system/server-overwatch.service`：

```ini
[Unit]
Description=Server Overwatch - GPU服务器监控程序
After=network.target

[Service]
Type=simple
User=gpu-monitor
Group=gpu-monitor
WorkingDirectory=/opt/server-overwatch
ExecStart=/opt/server-overwatch/.venv/bin/python main.py
Restart=always
RestartSec=10

# 日志输出
StandardOutput=journal
StandardError=journal
SyslogIdentifier=server-overwatch

[Install]
WantedBy=multi-user.target
```

### 2. 安装并启动服务

```bash
# 重新加载systemd配置
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable server-overwatch

# 启动服务
sudo systemctl start server-overwatch

# 检查服务状态
sudo systemctl status server-overwatch
```

## 管理命令

| 操作 | 命令 |
|------|------|
| 启动 | `sudo systemctl start server-overwatch` |
| 停止 | `sudo systemctl stop server-overwatch` |
| 重启 | `sudo systemctl restart server-overwatch` |
| 查看状态 | `sudo systemctl status server-overwatch` |
| 查看日志（实时） | `sudo journalctl -u server-overwatch -f` |
| 查看日志（最近100行） | `sudo journalctl -u server-overwatch -n 100` |
| 开机自启 | 已通过 `enable` 自动配置 |

## 故障排查

### 1. 服务启动失败

```bash
# 查看详细错误
sudo journalctl -u server-overwatch -xe

# 检查配置文件
cat /opt/server-overwatch/config.yaml
```

### 2. 机器人无法连接

- 确认 `config.yaml` 中的 `appid` 和 `secret` 正确
- 检查网络连通性

### 3. 数据采集失败

- 确认 `nvidia-smi` 等命令可用：`which nvidia-smi`
- 确认用户有执行权限

### 4. 日志文件权限问题

```bash
sudo chown -R gpu-monitor:gpu-monitor /opt/server-overwatch
```

## 文件结构

部署后的目录结构：

```
/opt/server-overwatch/
├── bot/                    # QQ机器人模块
├── monitor/                # 监控模块
├── utils/                  # 工具模块
├── wiki/                   # WIKI记忆库（可删除减小体积）
├── botpy/                  # QQ机器人SDK
├── main.py                 # 程序入口
├── config.yaml             # 配置文件
├── requirements.txt        # 依赖列表
├── .venv/                  # Python虚拟环境
└── logs/                   # 日志目录
    └── bot.log
```

## 注意事项

1. **安全建议**：使用专用用户运行，不要以root运行程序
2. **配置安全**：`config.yaml` 包含机器人密钥，确保文件权限适当
3. **日志管理**：日志默认输出到journald，可通过 `journalctl` 查看
4. **数据备份**：定期备份 `history.db` SQLite数据库
