## 🎯 核心结论与关键信息

### 功能需求

| 需求ID | 功能描述 | 优先级 | 状态 |
|--------|---------|--------|------|
| R1 | QQ机器人命令接收与解析 | P0 | ✅ 已设计 |
| R2 | 实时数据采集（GPU/CPU/内存/用户） | P0 | ✅ 已设计 |

### 架构设计

**Server Only** — 机器人程序直接运行在GPU服务器上，无需SSH功能。

| 项目 | 说明 |
|------|------|
| 采集间隔 | 每10分钟 |
| 数据保留 | 30天 |
| 数据库 | SQLite |
| 部署方式 | systemd |

### 命令设计

| 命令 | 功能 |
|------|------|
| `/info` | 查询当前所有在线用户资源使用 |
| `/info <用户名>` | 查询指定用户当前资源使用 |
| `/info 1d/1w/1m` | 查询历史记录 |
| `/help` | 帮助信息 |

## 📋 内容概述

本项目是一个运行在GPU服务器上的QQ机器人程序，用于监控服务器资源使用情况。用户通过QQ与机器人交互，可以查询当前在线用户的GPU、显存、CPU、内存使用情况，以及查看历史记录。
