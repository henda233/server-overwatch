# 📝 需求文档：历史查询命令翻页功能

> 创建时间：2026-04-08 20:40

## 需求描述

目前的某些命令会有多条数据，如果数据过多会被截断并显示「还有xxx条数据」。用户希望增加翻页功能来查看更多内容。

## 功能需求

### 1. 翻页语法
- `/info 5d 2` → 获取5天内，第2页的内容
- `/info 5d` → 默认为第1页
- 所有翻页参数作为最后一个参数

### 2. 适用范围
所有历史查询命令均需支持翻页：

| 命令 | 翻页后格式 |
|------|-----------|
| `/info <时间>` | `/info 5d [页码]` |
| `/info <用户> <时间>` | `/info 张三 5d [页码]` |
| `/stats <时间>` | `/stats 1w [页码]` |
| `/top [类型]` | `/top gpu [页码]` |
| `/users [天数]` | `/users 7d [页码]` |

### 3. 配置项
- 默认每页10条
- 可在 `config.yaml` 中调整 `pagination.page_size`

### 4. 导航信息格式

**第1页输出：**
```
📊 张三 - 历史记录（5天内）
第1页/共4页 | 还有3页数据，输入 /info 张三 5d 2 查看下一页
─────────────────────────────
10:00 | GPU: 50% | 显存: 8GB | CPU: 20% | 内存: 15%
...
```

**最后一页输出：**
```
📊 张三 - 历史记录（5天内）
第4页/共4页（最后一页）
─────────────────────────────
10:00 | GPU: 50% | 显存: 8GB | CPU: 20% | 内存: 15%
```

### 5. 边界处理
- 页码超出范围 → 静默返回最后一页
- 非数字页码（如 `/info 5d abc`）→ 静默忽略，返回第1页
