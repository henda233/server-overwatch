# 🏗️ 系统架构设计

> PLAN-001 | v2.0 | 2026-04-07 | status: completed

## 1. 架构设计理念

**Server Only** — 机器人程序直接运行在GPU服务器上，无需SSH功能。用户与QQ机器人对话，直接与本地程序交互获取结果。

```
┌─────────────────────────────────────────────────────┐
│                    GPU 服务器                        │
│  ┌─────────────────────────────────────────────┐    │
│  │              systemd 服务                    │    │
│  │  ┌─────────────────────────────────────┐    │    │
│  │  │           main.py (主进程)           │    │    │
│  │  │                                     │    │    │
│  │  │  ┌───────────┐  ┌───────────────┐   │    │    │
│  │  │  │ bot 模块   │  │ monitor 模块  │   │    │    │
│  │  │  │           │  │               │   │    │    │
│  │  │  │ handler   │  │ collector     │   │    │    │
│  │  │  │           │  │ recorder      │   │    │    │
│  │  │  │           │  │ formatter     │   │    │    │
│  │  │  └───────────┘  └───────────────┘   │    │    │
│  │  │                                     │    │    │
│  │  │        ┌─────────────────┐           │    │    │
│  │  │        │   SQLite DB    │           │    │    │
│  │  │        │  (history.db)  │           │    │    │

└─────────────────────────────────────────────────────┘
          ↑ QQ协议通信
          ↓
    ┌─────────────┐
    │   QQ 用户   │
    └─────────────┘
```

## 2. 模块职责

| 模块 | 职责 |
|------|------|
| `config.yaml` | 机器人凭证、采集间隔、数据保留天数 |
| `main.py` | 启动机器人、调度定时采集任务 |
| `bot/handler.py` | 接收QQ消息、解析命令、返回结果 |
| `monitor/collector.py` | 采集GPU/CPU/内存/用户在线状态 |
| `monitor/recorder.py` | SQLite存储、30天自动清理 |
| `monitor/formatter.py` | 查询结果格式化为表格 |
| `utils/config.py` | 加载YAML配置 |

## 3. 目录结构

```
server-overwatch/
├── config.yaml              # 配置文件
├── main.py                  # 程序入口
├── bot/                     # QQ机器人模块
│   └── handler.py           # 命令处理
├── monitor/                 # 系统监控模块
│   ├── collector.py         # 实时数据采集
│   ├── recorder.py          # 历史记录存储
│   └── formatter.py         # 格式化输出
├── utils/                   # 工具模块
│   └── config.py            # 配置加载
└── requirements.txt         # 依赖
```

## 4. 数据流向

```
用户: "@机器人 /info"
        │
        ▼
bot/handler.py
  - 解析命令 (/info)
  - 判断实时 or 历史
        │
        ▼
┌─────────────────────────────────┐
│  实时查询 (/info)               │
│  monitor/collector.py           │
│  - nvidia-smi → GPU/显存       │
│  - free → 内存                 │
│  - top → CPU                   │
│  - who → 在线用户              │
└─────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────┐
│  历史查询 (/info 1d/1w/1m)      │
│  monitor/recorder.py           │
│  - SQLite查询                   │
└─────────────────────────────────┘
        │
        ▼
monitor/formatter.py → 表格输出
        │
        ▼
回复用户
```

## 5. 关键技术选型

| 技术 | 选型 | 说明 |
|------|------|------|
| QQ机器人SDK | botpy | 官方Python SDK |
| 数据库 | SQLite | 轻量，无需额外服务 |
| 配置管理 | pyyaml | YAML配置解析 |
| 定时任务 | asyncio + threading | 定时采集数据 |
| 系统命令 | subprocess | 执行nvidia-smi等 |

## 6. 与旧架构对比

| 项目 | 旧架构 (v1.0) | 新架构 (v2.0) |
|------|---------------|---------------|
| 部署方式 | SSH远程连接 | Server Only本地运行 |
| SSH模块 | 需要 | **移除** |
| 数据采集 | 实时SSH执行 | 本地采集 + 定时入库 |
| 复杂度 | 高 | **低** |
| 响应速度 | 依赖网络 | **更快** |
